# ReDist Printable Meeting Pack

Purpose: PDF-ready print and digital meeting package assembled from the executive one-pager, portfolio, meeting packs, discovery questions, scorecard, and pilot qualification checklist.  
Recommended format: A4 portrait PDF.  
Recommended length: 18-28 pages depending on audience.  
QR destination: `https://www.redistribut.com`

## Print Brand Rules

- Cover: ReDist wordmark top-left, UAE Pilot Edition label top-right.
- Footer: small ReDist mark, website URL, page number.
- Colors: brand-900 headings, brand-700 accents, neutral text, brand-50 section backgrounds.
- Card radius: 8px.
- Typography: Inter with Arabic fallback `IBM Plex Sans Arabic` or `Noto Sans Arabic`.
- English first, Arabic directly underneath when customer-facing.
- QR code size: 24-28 mm on cover and final page; 15-18 mm in footer or appendix.
- QR caption: `Scan to visit redistribut.com`.

## Universal Page Structure

### Page 1: Cover

Elements:

- ReDist wordmark.
- Audience label: Executive / Supplier / Recipient / Partner.
- UAE Pilot Edition.
- Founder name placeholder.
- Website and contact placeholders.
- QR code bottom-right.

Image placeholder:

- Website hero screenshot or approved workspace screenshot.

### Page 2: Why This Matters

Problem statement:

Useful surplus should not disappear into informal coordination.

Design:

- One sentence headline.
- Three operational pain cards.
- Small UAE Pilot Edition note.

### Page 3: ReDist Solution

Workflow:

List -> Discover -> Request -> Approve -> Handover -> Verify -> Certificate -> Impact

Design:

- Horizontal workflow on landscape export.
- Vertical numbered steps on A4 portrait.

### Page 4: Trust Boundary

Include:

- Verified organization workflow.
- Supplier approval control.
- Handover verification.
- Certificate-backed operational evidence.
- Certificate limitation note.

### Page 5: Pilot Model

Include:

- One supplier.
- One recipient.
- One low-risk category.
- One transfer.
- One certificate.
- One feedback cycle.

### Page 6: Audience-Specific Value

Supplier:

- Publish surplus.
- Review requests.
- Control handover.
- Preserve evidence.

Recipient:

- Discover resources.
- Submit requests.
- Track handover.
- Review evidence.

Partner:

- Introduce networks.
- Support ESG evidence.
- Build category playbooks.
- Support UAE ecosystem.

### Page 7: Discovery Questions

Use 8-12 selected questions from `REDIST_DISCOVERY_QUESTION_LIBRARY.md`.

### Page 8: Qualification Checklist

Use the relevant section from `REDIST_PILOT_QUALIFICATION_CHECKLIST.md`.

### Page 9: Meeting Scorecard

Use `REDIST_FOUNDER_MEETING_EVALUATION_FORM.md`.

### Final Page: Contact And QR

Elements:

- ReDist wordmark.
- Founder contact placeholder.
- Website.
- QR linking to `https://www.redistribut.com`.
- CTA: `Request a pilot conversation`.

- Certificate view.
- Impact dashboard.
- UAE city or operational site photo only if approved and non-generic.

## PDF Export Recommendations

- Export as PDF/A where possible for sharing.
- Keep file under 10 MB for email.
- Use compressed screenshots.
- Create separate filenames:
  - `ReDist_Executive_One_Pager_UAE_Pilot.pdf`
  - `ReDist_Supplier_Meeting_Pack_UAE_Pilot.pdf`
  - `ReDist_Recipient_Meeting_Pack_UAE_Pilot.pdf`
  - `ReDist_Partnership_Meeting_Pack_UAE_Pilot.pdf`
- Do not include confidential pilot participant names without approval.
- Do not include unapproved impact metrics.
- Add version date in footer.

## Presentation And Print Flow

1. Open with the one-page summary.
2. Explain problem and workflow.
3. Confirm audience role.
4. Review trust and evidence boundary.
5. Ask discovery questions.
6. Complete qualification checklist.
7. Score fit.
8. Agree next step.
