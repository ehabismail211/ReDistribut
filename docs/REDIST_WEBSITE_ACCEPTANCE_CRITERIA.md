# ReDist Website Acceptance Criteria

Date: 2026-06-22  
Status: Launch acceptance criteria for marketing website  
Scope: Desktop, mobile, performance, SEO, analytics, lead capture

## Launch Gate

The marketing website is ready for soft launch when all Must Have criteria below pass.

Soft launch means:

- URL can be shared in founder outreach.
- Lead capture works.
- Claims are reviewed.
- Mobile experience is usable.

It does not mean broad public launch or paid acquisition.

## Desktop Criteria

Must pass:

- Home page loads without visual breakage.
- Navigation links work.
- Hero message is visible above fold.
- Role CTAs route correctly.
- All MVP pages render without missing content.
- FAQ accordion or sections are usable.
- Contact form is readable and complete.
- Footer links work.

Should pass:

- Layout feels polished at common desktop widths.
- No text overlaps.
- CTA hierarchy is obvious.

## Mobile Criteria

Must pass:

- Navigation is usable on mobile.
- CTA buttons are tappable.
- Text does not overflow containers.
- FAQ is usable by touch.
- Contact form is easy to complete.
- No horizontal scrolling.
- All sections stack cleanly.

Should pass:

- Hero communicates value without excessive scrolling.
- Contact form intent is preserved from CTA links.

## Performance Criteria

Must pass:

- Pages load quickly on standard mobile connection.
- Images are optimized.
- No large unused assets.
- No blocking third-party scripts beyond selected analytics.

Suggested targets:

- Lighthouse Performance: 85+.
- Largest Contentful Paint: under 2.5s where practical.
- Cumulative Layout Shift: under 0.1.

## SEO Criteria

Must pass:

- Unique title for each page.
- Unique meta description for each page.
- Semantic H1 on each page.
- Open Graph title and description.
- Sitemap generated if implementation supports it.
- FAQ content indexable.

Target keywords:

- surplus redistribution UAE.
- circular economy UAE.
- verified resource redistribution.
- surplus resource handover.
- ESG redistribution evidence.

## Analytics Criteria

Must pass:

- Page views tracked.
- CTA clicks tracked.
- Contact form starts tracked.
- Contact form submissions tracked.
- Lead intent captured.
- UTM parameters captured where possible.

Events:

- `cta_click_request_pilot`
- `cta_click_supplier`
- `cta_click_recipient`
- `cta_click_partner`
- `cta_click_demo`
- `contact_form_start`
- `contact_form_submit`

## Lead Capture Criteria

Must pass:

- Contact form captures required fields.
- Form does not request sensitive documents.
- Consent checkbox exists.
- Success state appears after submit.
- Founder can access submitted lead.
- Lead source/intent is visible.
- Test submission is completed before launch.

Required fields:

- Full name.
- Work email.
- Organization.
- Role.
- City / emirate.
- Interest type.
- Organization type.
- Resource category or need.
- Consent to contact.

## Public Claims Criteria

Must pass:

- No real pilot results claimed before real evidence.
- No audited ESG claim.
- No government approval claim.
- No legal, tax, safety, or donation certification claim.
- Certificate limitation is visible where certificates are discussed.
- Pilot status is clear.
