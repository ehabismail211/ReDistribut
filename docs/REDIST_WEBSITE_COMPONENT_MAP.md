# ReDist Website Component Map

Date: 2026-06-22  
Status: Marketing website component handoff  
Scope: Reusable public website components

## Component Principle

Components should support clarity, trust, and conversion. Avoid decorative complexity that distracts from the founder-guided pilot message.

## Hero

Used on:

- Home.
- How It Works.
- Suppliers.
- Recipients.
- Impact.
- FAQ.
- Contact.

Props:

- Eyebrow.
- Headline.
- Subheadline.
- Primary CTA label.
- Primary CTA href.
- Secondary CTA label.
- Secondary CTA href.
- Trust note.

Rules:

- Homepage hero should include the clearest ReDist positioning.
- Inner page heroes should be shorter.
- Do not use fake logos, fake metrics, or fake case study numbers.

## Feature Grid

Used on:

- Home.
- How It Works.
- Suppliers.
- Recipients.

Feature items:

- Resource listings.
- Resource discovery.
- Request workflow.
- Handover tracking.
- Certificate evidence.
- Impact visibility.

Props:

- Icon.
- Title.
- Description.
- Optional link.

Rules:

- Keep copy operational.
- Avoid generic SaaS buzzwords.

## Impact Cards

Used on:

- Home.
- Impact.

Cards:

- AED value recovered.
- Items redistributed.
- Waste diverted.
- Certificate evidence.
- Recipient usefulness.
- Environmental estimates.

Props:

- Title.
- Description.
- Status label.
- Evidence note.

Rules:

- Use capability labels, not fake results.
- Example status labels: "Designed to track", "Requires completed transfer", "Estimate only".

## FAQ Accordion

Used on:

- FAQ.
- Optional homepage FAQ preview.

Props:

- Question.
- Answer.
- Category.
- Default open state.

Rules:

- Keyboard accessible.
- Answers should be readable without relying on icons.
- Prioritize certificate, pilot status, category, and verification questions.

## Partner Logos

Used on:

- Partners.
- Home, after real partner approvals exist.

Current rule:

- Do not display logos until written approval is received.
- Before approval, use category labels only: NGOs, hotels, restaurants, warehouses, ESG partners.

Props:

- Organization name.
- Logo asset.
- Approval status.
- Link.

## Contact Form

Used on:

- Contact.

Fields:

- Full name.
- Work email.
- Organization name.
- Role / title.
- City / emirate.
- Interest type.
- Organization type.
- Resource category or need.
- Timeline.
- Message.
- Consent to contact.

Rules:

- Preserve URL intent query.
- Do not collect sensitive documents.
- Show clear success state.
- Route to founder review.

## CTA Blocks

Used on:

- Every page.

Variants:

- General pilot CTA.
- Supplier CTA.
- Recipient CTA.
- Partner CTA.
- Impact/ESG CTA.
- FAQ/contact CTA.

Props:

- Headline.
- Body.
- Primary CTA.
- Secondary CTA.
- Context label.

Rules:

- CTAs should ask for conversations, not public signups.
- Avoid "join now" or "start redistributing immediately" until self-serve launch exists.

## Trust Indicators

Used on:

- Home.
- Suppliers.
- Recipients.
- Impact.
- Contact.

Indicators:

- Verified organization workflow.
- Request approval before handover.
- Certificate-backed completion evidence.
- Protected pilot access.
- Founder-guided onboarding.
- Clear impact boundaries.

Rules:

- Trust indicators must be factual.
- Do not imply legal, safety, or audited ESG certification.
