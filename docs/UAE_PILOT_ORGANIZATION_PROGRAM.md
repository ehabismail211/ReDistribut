# UAE Pilot Organization Program

Date: 2026-06-20

## Purpose

This program defines the onboarding, verification, workflow expectations, KPI targets, and feedback requirements for the first UAE pilot organization cohorts.

Pilot organization types:

1. Restaurant
2. Hotel
3. Warehouse
4. Packaging Supplier
5. NGO

No implementation was performed for this document.

## Program Principles

- Invite-only pilot.
- UAE-first operating model.
- Limited categories and manual oversight.
- Verification before sensitive or regulated listings.
- Every completed transfer must produce transaction, handover, impact, trust, and certificate evidence.
- Feedback is mandatory after each first completed transfer.

## 1. Restaurant

### Onboarding Workflow

1. Founder invites restaurant owner/admin.
2. Organization registers and completes profile.
3. Restaurant submits verification.
4. Founder reviews trade license and food permit.
5. Restaurant creates first approved listing.
6. Restaurant completes assisted first transfer.
7. Founder reviews handover, certificate, impact, and feedback.

### Verification Requirements

| Requirement | Required |
| --- | --- |
| Trade license | Yes |
| Food permit | Yes |
| Authorized contact | Yes |
| Registered address | Yes |
| Same-day handover SOP acknowledgement | Yes |

### Success Criteria

- Restaurant publishes at least 2 approved listings.
- At least 1 listing is requested by an approved recipient.
- At least 1 transfer is completed without over-allocation.
- Certificate and QR verification are generated and reviewed.
- No unresolved Critical or High issues remain after first transfer.

### Expected Listings

- Prepared meals near expiry.
- Packaged food surplus.
- Kitchen consumables suitable for redistribution.

### Expected Requests

- NGO meal request.
- School or community organization request.

### Expected Transactions

| Target | Value |
| --- | ---: |
| Completed transfers | 1-2 |
| Average completion window | Same day |
| Certificate coverage | 100% |

### KPI Targets

| KPI | Target |
| --- | ---: |
| Verification completion | 100% |
| Listing creation | 2+ |
| Request acceptance rate | 70%+ |
| Handover completion | 1+ |
| Waste prevented | 25 kg+ |
| Feedback submission | 100% after first transfer |

### Feedback Requirements

- Ease of verification.
- Clarity of food category restrictions.
- Request management friction.
- Same-day handover feasibility.
- Certificate usefulness.

## Weekly Founder Review

Each week, review:

1. Organization onboarding status.
2. Verification completion.
3. Listings created.
4. Requests created.
5. Transactions completed.
6. Handover issues.
7. Certificates generated.
8. Impact metrics.
9. Trust score changes.
10. Feedback and issue severity.

## Stop Conditions

Pause onboarding if:

1. Any cross-tenant access issue is found.
2. Any private document or evidence exposure occurs.
3. A regulated category is listed without approval.
4. Two or more handovers fail for the same workflow reason.
5. Certificate public QR exposes sensitive internal information.
6. Mobile browser workflow blocks transaction completion for pilot users.
