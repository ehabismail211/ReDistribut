# ReDist Administrator Guide

Date: 2026-06-21  
Status: Founder/admin operations guide  
Audience: Founder, Platform Admin, Reviewer, future Operations Lead

## Purpose

This guide defines admin responsibilities and daily admin workflows for ReDist.

## Admin Responsibilities

Administrators are responsible for:

- Reviewing organization verification.
- Monitoring trust and category eligibility.
- Reviewing audit events.
- Handling moderation reports.
- Monitoring certificates and disputes.
- Escalating Critical/High issues.
- Preserving participant trust and tenant boundaries.

Admins must not:

- Access participant data without operational reason.
- Approve restricted categories without founder/legal approval.
- Treat certificates as legal/tax/government documents.
- Ignore data/privacy/security concerns.
- Expand pilot access without founder approval.

## Verification Process

Steps:

1. Confirm organization identity and primary contact.
2. Review submitted profile details.
3. Review documents or founder-approved manual evidence.
4. Check document expiry and relevance.
5. Confirm category eligibility.
6. Approve, reject, or request changes.
7. Record decision and notes.
8. Escalate unclear legal/category issues.

Verification outcomes:

- Pending review.
- Approved for low-risk pilot category.
- Request changes.
- Rejected.
- Suspended or restricted.

## Trust Review

Trust review should consider:

- Verification status.
- Completed transfers.
- Failed handovers.
- Cancellations or disputes.
- Response behavior.
- Audit events.
- Document expiry.
- Profile completeness.

Trust score should be treated as an operational confidence signal, not a public marketplace rating.

Escalate if:

- Trust drops after dispute.
- Document expires for active category.
- Organization repeatedly fails handovers.
- Organization attempts unsafe or restricted listing.

## Audit Review

Review audit logs for:

- Permission denied events.
- Role assignment/revocation.
- Verification decisions.
- Request and transfer actions.
- Certificate events.
- Admin actions.
- Suspicious access patterns.

Actions:

- Investigate unexpected denied events.
- Confirm privileged actions have clear reason.
- Preserve audit records for disputes.
- Escalate suspected unauthorized access.

## Moderation Review

Moderation applies to:

- Unsafe listings.
- Restricted categories.
- Misleading resource details.
- Suspicious requests.
- Abusive behavior.
- Certificate disputes.
- Public verification issues.

Moderation actions:

- Request clarification.
- Hide or pause workflow.
- Restrict category.
- Escalate to founder.
- Mark issue resolved or deferred.
- Preserve evidence.

## Certificate Review

Admins should verify:

- Certificate matches completed transfer.
- Supplier and recipient are correct.
- Quantity and resource are correct.
- Date/location are correct.
- QR verification works.
- Public-safe data is limited.

If disputed:

- Pause external sharing.
- Log issue.
- Review transfer evidence.
- Correct, revoke, or escalate according to framework.

## Admin Daily Checklist

| Area | Check |
| --- | --- |
| Verification | New submissions, pending documents, expiry risks |
| Requests | Requests awaiting supplier action |
| Transfers | Accepted handovers, verification required |
| Certificates | Missing, mismatched, disputed certificates |
| Audit | Denied permissions, privileged actions |
| Moderation | Reports, unsafe categories, suspicious activity |
| Issues | Critical/High owner and target date |

## Admin Conduct

Admins should:

- Use minimum necessary access.
- Record decisions clearly.
- Avoid informal approvals outside the tracker.
- Escalate uncertainty.
- Respect confidentiality.
- Preserve audit evidence.
