# ReDist Weekly Operations Runbook

Date: 2026-06-21  
Cadence: Weekly during pilot and early commercial operation  
Owner: Founder during pilot; Operations Lead after first hire

## Purpose

Use this runbook to review operating health, risks, participant progress, and launch readiness every week.

## Weekly Review Agenda

| Step | Area | Output |
| --- | --- | --- |
| 1 | Metrics review | Updated success metrics |
| 2 | Risk review | Updated risk register |
| 3 | Pilot review | Continue / Expand / Pause recommendation |
| 4 | Participant review | Participant-by-participant status |
| 5 | Escalation review | Owner and next action for unresolved issues |
| 6 | Launch blocker review | Updated blocker status |

## 1. Metrics Review

Review:

- Organizations invited.
- Organizations onboarded.
- First logins completed.
- Listings created.
- Requests submitted.
- Requests approved.
- Transfers completed.
- Certificates generated.
- Feedback submitted.
- Satisfaction ratings.

Questions:

- Are participants moving through the funnel?
- Are transfers completing?
- Are certificates generated for 100% of completed transfers?
- Is feedback positive enough to continue?

Output:

- Update `docs/UAE_PILOT_SUCCESS_METRICS.md`.

## 2. Risk Review

Review:

- Top 10 launch risks.
- Open Critical/High issues.
- Legal/privacy blockers.
- Security incidents.
- Certificate disputes.
- Restricted category concerns.

Questions:

- Did any risk change severity?
- Is any mitigation no longer sufficient?
- Does any risk block expansion or paid launch?

Output:

- Update `docs/REDIST_RISK_REGISTER_V1.md` if risk state changed.

## 3. Pilot Review

Review:

- Tracking center.
- Issue log.
- Feedback forms.
- First transfer post-mortem.
- Decision gate.

Questions:

- Should the pilot continue?
- Should the cohort expand?
- Should the pilot pause and fix?
- Is founder support load acceptable?

Output:

- Update `docs/UAE_PILOT_DECISION_GATE.md`.

## 4. Participant Review

For each participant:

- Current status.
- Last action completed.
- Next action.
- Open issues.
- Feedback status.
- Trust or verification concerns.
- Expansion suitability.

Output:

- Update `docs/UAE_PILOT_WAVE1_TRACKING_CENTER.md`.

## 5. Escalation Review

Review:

- Critical issues.
- High issues.
- Aging Medium issues.
- Disputes.
- Certificate problems.
- Support delays.

Actions:

- Assign owner.
- Set target resolution date.
- Decide if workflow must pause.
- Decide if founder/legal/security review is needed.

Output:

- Updated issue log and escalation owners.

## 6. Launch Blocker Review

Review:

- Product blockers.
- Pilot blockers.
- Commercial blockers.
- Legal blockers.
- Security/compliance blockers.

Output:

- Update `docs/REDIST_LAUNCH_BLOCKERS_REGISTER.md`.
- Reconfirm launch recommendation.

## Weekly Decision Format

| Decision | Meaning |
| --- | --- |
| Continue | Keep current scope and guided operations |
| Expand | Add approved participants or workflows |
| Pause and fix | Stop expansion or affected workflow |
| No-Go | Do not proceed until blocker closes |

## Weekly Closeout

End every weekly review with:

- One decision.
- Top 3 priorities.
- Owners.
- Target dates.
- Risks accepted by founder, if any.
