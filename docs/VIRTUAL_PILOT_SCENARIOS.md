# ReDist Virtual Pilot Scenarios

Date: 2026-06-22  
Status: Simulated transfer scenario set  
Evidence level: Virtual rehearsal only. These scenarios are not completed real-world transfers.

## Simulation Boundary

These 10 scenarios are designed to stress-test ReDist's workflow using realistic UAE operating situations. Outcomes must be recorded in [Virtual Pilot Results Log](VIRTUAL_PILOT_RESULTS_LOG.md) and should never be described as live pilot success.

## VP-01: Urgent Sealed Meal Pack Handover

Assumption:

- Prepared food workflows require compliance review. This simulation uses sealed meal packs and assumes pilot-safe handling conditions for rehearsal only.

Workflow:

1. Supplier A creates an urgent listing for 80 sealed meal packs expiring within 24 hours.
2. Recipient A submits a request for all 80 packs.
3. Supplier A reviews recipient details and approves.
4. Recipient A confirms same-day pickup window.
5. Supplier A marks handover ready.
6. Recipient A verifies receipt.
7. Certificate is generated and reviewed.

Expected learning:

- Whether urgency is visible.
- Whether pickup timing is clear.
- Whether certificate language avoids food-safety overclaiming.

## VP-02: Pantry Cartons Partial Request

Workflow:

1. Supplier A lists 100 sealed pantry cartons.
2. Recipient A requests 40 cartons.
3. Supplier A approves partial transfer.
4. Remaining quantity stays visible.
5. Transfer completes and certificate reflects 40 cartons only.

Expected learning:

- Quantity tracking.
- Partial transfer clarity.
- Impact calculations based on completed quantity.

## VP-03: Reusable Service Trays

Workflow:

1. Supplier A lists 25 reusable service trays after an event equipment refresh.
2. Recipient B requests 20 trays for training events.
3. Supplier A approves.
4. Recipient B asks whether the remaining 5 can be included.
5. Founder observes whether the workflow supports quantity clarification without confusion.

Expected learning:

- Suitability language.
- Request editing or follow-up limitations.
- Whether notes are enough for operational clarification.

## VP-04: Banquet Chairs

Workflow:

1. Supplier B lists 30 banquet chairs with condition notes and photos if available.
2. Recipient B requests 30 chairs.
3. Supplier B approves after confirming pickup capacity.
4. Handover is scheduled at hotel loading bay.
5. Recipient B verifies receipt and condition.

Expected learning:

- Condition transparency.
- Pickup location clarity.
- Certificate usefulness for non-food furniture transfers.

## VP-05: Towel Sets

Workflow:

1. Supplier B lists 120 clean towel sets as hospitality surplus.
2. Recipient A requests 80 sets.
3. Supplier B approves 80 and leaves 40 available.
4. Recipient A requests certificate wording that confirms receipt, not product guarantee.

Expected learning:

- Condition and trust language.
- Certificate expectation management.
- Remaining quantity visibility.

## VP-06: Shelving Units

Workflow:

1. Supplier B lists 8 shelving units from a storage-room renovation.
2. Recipient B requests 4 units.
3. Supplier B approves with dismantling and pickup notes.
4. Recipient B confirms transport availability.
5. Handover completes after scheduled pickup.

Expected learning:

- Bulky resource instructions.
- Pickup readiness.
- Whether "next step" language is clear.

## VP-07: Plastic Storage Bins

Workflow:

1. Supplier C lists 250 plastic storage bins.
2. Recipient B requests 150.
3. Supplier C approves.
4. Recipient B verifies count at pickup.
5. Certificate records completed transfer quantity.

Expected learning:

- Bulk quantity verification.
- Transfer completion confidence.
- Certificate quantity clarity.

## VP-08: Cardboard Cartons

Workflow:

1. Supplier C lists 500 clean cardboard cartons.
2. Recipient A requests 300.
3. Recipient B attempts to request 250 before the first approval completes.
4. Founder checks whether over-allocation is prevented or operationally obvious.

Expected learning:

- Reservation clarity.
- Competing request handling.
- Quantity availability communication.

## VP-09: Reusable Pallets

Workflow:

1. Supplier C lists 60 reusable pallets.
2. Recipient B requests all 60.
3. Supplier C approves only after truck access is confirmed.
4. Handover requires loading dock instructions.
5. Recipient B verifies receipt.
6. Certificate is reviewed for logistics evidence.

Expected learning:

- Logistics-only workflow fit.
- Verification step clarity.
- Whether certificate supports warehouse partner confidence.

## VP-10: Parallel Requests Queue

Workflow:

1. Supplier A creates one urgent listing.
2. Supplier B creates one furniture listing.
3. Recipient A requests Supplier A's listing.
4. Recipient B requests Supplier B's listing.
5. Founder reviews Dashboard, Requests, Transfers, and Impact to monitor both.

Expected learning:

- Operational queue clarity.
- Founder monitoring burden.
- Whether active work is easy to distinguish from completed transfers.

## Scenario Completion Rule

A scenario is considered virtually complete only when:

- Listing is created.
- Request is submitted.
- Supplier decision is recorded.
- Handover status is reviewed.
- Verification status is reviewed.
- Certificate or certificate limitation is documented.
- Issues are logged with severity.
