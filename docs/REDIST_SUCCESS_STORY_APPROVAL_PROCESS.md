# ReDist Success Story Approval Process

Date: 2026-06-22  
Status: Approval workflow for public case studies and success stories  
Purpose: Protect participant trust, privacy, legal safety, and evidence integrity.

## Approval Principle

No ReDist success story should be published until:

- The transfer is completed.
- Evidence is collected.
- Metrics are reviewed.
- Sensitive data is redacted.
- Supplier approval is received.
- Recipient approval is received.
- Founder approves publication.
- Legal/privacy review is completed if needed.

## Consent Requirements

Required consent:

| Consent Type | Supplier | Recipient |
| --- | --- | --- |
| Use organization name | Required | Required |
| Use logo | Required | Required |
| Use quote | Required | Required |
| Use photo | Required if identifiable | Required if identifiable |
| Use certificate reference | Required | Required |
| Use transfer metrics | Required | Required |
| Use anonymized story | Recommended | Recommended |

Consent format:

- Written email approval is acceptable for pilot stories.
- Formal release may be required for public campaigns, press, or investor decks.

## Organization Approval

Approval steps:

1. Draft case study internally.
2. Redact sensitive information.
3. Send supplier-specific section to supplier.
4. Send recipient-specific section to recipient.
5. Ask each organization to approve:
   - Name use.
   - Quote use.
   - Metrics.
   - Images.
   - Certificate reference.
6. Record approval date and approver.
7. Founder reviews final version.
8. Publish only approved version.

## Redaction Rules

Always redact unless explicitly approved:

- Personal names.
- Email addresses.
- Phone numbers.
- Private messages.
- Internal IDs.
- Exact pickup addresses.
- Sensitive document details.
- Financial values if not approved.
- Photos with people, vehicle plates, documents, or private areas.

Do not publish:

- Unresolved disputes.
- Negative feedback without anonymization and context.
- Security or access-control details.
- Legal, safety, or tax interpretations.
- Claims that certificate is audited ESG proof.

## Publication Approval Workflow

| Step | Owner | Output |
| --- | --- | --- |
| Evidence collected | Founder / Operations | Evidence packet |
| Case study drafted | Founder | Draft case study |
| Metrics reviewed | Founder | Metrics notes |
| Redaction completed | Founder / Operations | Public-safe draft |
| Supplier approval | Supplier contact | Written approval |
| Recipient approval | Recipient contact | Written approval |
| Legal/privacy review if needed | Advisor / Founder | Approval or edits |
| Founder final approval | Founder | Publish decision |
| Publication | Founder / Marketing | Website, deck, or internal use |

## Publication Levels

| Level | Description | Approval Needed |
| --- | --- | --- |
| Internal only | Used for learning and operations | Founder |
| Anonymized external | No organization names or identifying details | Founder plus participant consent |
| Named case study | Organization names included | Supplier and recipient written approval |
| Press / public launch | Broad public use | Written approval plus legal/privacy review |
| Investor / sponsor deck | Fundraising or sponsorship material | Written approval for all named claims |
