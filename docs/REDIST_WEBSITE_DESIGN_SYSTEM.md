# ReDist Website Design System

Date: 2026-06-22  
Status: Lightweight public website design system  
Scope: Marketing website only

## Design Direction

The ReDist marketing website should feel:

- Trustworthy.
- Operational.
- Modern.
- UAE-first.
- Enterprise-ready.
- Practical rather than decorative.

It should not feel like:

- A consumer marketplace.
- A charity donation page only.
- A crypto/token marketplace.
- A generic startup landing page.
- An audited ESG certification provider.

## Colors

Recommended palette:

| Token | Color | Usage |
| --- | --- | --- |
| `ink` | `#17201B` | Primary text |
| `muted-ink` | `#53605A` | Secondary text |
| `canvas` | `#F7F5EF` | Page background |
| `surface` | `#FFFFFF` | Cards and form surfaces |
| `line` | `#D9DED8` | Borders and dividers |
| `green` | `#2F6B4F` | Primary brand / CTA |
| `green-dark` | `#214B38` | Hover / emphasis |
| `teal` | `#2E7C83` | Links and secondary accents |
| `gold` | `#B9852C` | Impact highlights |
| `red` | `#B44A3C` | Warnings / high-risk notices |

Rules:

- Use green for action and trust.
- Use gold sparingly for impact evidence.
- Use red only for cautions and risk notes.
- Keep backgrounds calm and readable.

## Typography

Recommended approach:

- Use a clean sans-serif font for all text.
- Use strong hierarchy, not oversized decorative type.
- Keep body copy highly readable.

Suggested scale:

| Element | Size Guidance |
| --- | --- |
| H1 | 48-64 desktop, 36-42 mobile |
| H2 | 32-40 desktop, 28-32 mobile |
| H3 | 22-26 |
| Body | 16-18 |
| Small / notes | 14-15 |

Rules:

- No negative letter spacing.
- No tiny low-contrast legal notes.
- Keep public disclaimers readable.

## Buttons

Button variants:

| Variant | Usage |
| --- | --- |
| Primary | Main CTA, green background |
| Secondary | Outline CTA |
| Text link | Low-emphasis navigation |
| Warning link | Risk or boundary details |

Button copy examples:

- Request a pilot conversation.
- Become a pilot supplier.
- Become a pilot recipient.
- See how it works.
- Partner with ReDist.

Rules:

- Avoid "Buy", "Shop", "Cart", or "Sign up instantly".
- Buttons should clearly state next action.
- Mobile buttons should be full-width where helpful.

## Cards

Card usage:

- Feature blocks.
- Workflow steps.
- Impact metric capabilities.
- FAQ groups.
- Partner types.

Rules:

- Use cards for repeated content, not every section.
- Keep card radius modest.
- Avoid card-inside-card layouts.
- Keep card text concise.

## Icons

Recommended icon themes:

- List.
- Search.
- Clipboard / request.
- Handover / arrows.
- Shield / verification.
- Certificate.
- Chart / impact.
- Building / organization.

Rules:

- Icons should clarify, not decorate.
- Avoid shopping icons.
- Avoid charity-only iconography that narrows positioning.

## Trust Indicators

Trust indicator styling:

- Use compact badges or rows.
- Pair icon with short label.
- Keep explanation nearby if claim has boundary.

Recommended labels:

- Verified organization workflow.
- Supplier approval control.
- Handover verification.
- Certificate evidence.
- Impact visibility.
- Founder-guided pilot.

Boundary labels:

- "Operational evidence, not legal certification."
- "Impact estimates require completed transfers."
- "Pilot access is controlled."

## Forms

Contact form design:

- Single-column on mobile.
- Clear labels.
- Required fields marked plainly.
- Helpful placeholder text.
- Confirmation state after submit.

Required form note:

Do not submit sensitive documents through this form.

## Accessibility

Minimum expectations:

- Semantic headings.
- Keyboard accessible navigation.
- Keyboard accessible FAQ accordion.
- Form labels connected to inputs.
- Visible focus states.
- Color contrast reviewed.
- No text embedded only in images.
