# ReDist Website Sitemap

Date: 2026-06-22  
Status: Public launch website planning draft  
Evidence boundary: Public copy must not claim real pilot results until live workflows are completed and approved for publication.

## Website Goal

The public website should explain ReDist clearly to UAE suppliers, recipients, partners, and ESG stakeholders without overclaiming traction.

Primary website objective:

- Convert qualified organizations into founder-led pilot or partnership conversations.

Secondary website objectives:

- Explain the redistribution workflow.
- Build trust around verification, handover evidence, and certificates.
- Show the future impact vision while labeling current status honestly.
- Route suppliers, recipients, and partners to the right inquiry path.

## Page Relationships

Home should route users by intent:

- "I have surplus resources" -> Suppliers.
- "I need resources" -> Recipients.
- "I want to partner or sponsor" -> Partners.
- "I need to understand evidence and reporting" -> Impact.

How It Works should route users by workflow:

- List -> Suppliers.
- Discover and Request -> Recipients.
- Verify and Impact -> Partners / ESG stakeholders.

FAQ should reduce friction before outreach:

- What is ReDist?
- Is this a marketplace?
- What counts as a certificate?
- Is the pilot public?
- What categories are allowed?
- How does verification work?
