# ReDist Product Summary

## Original Idea

The initial `Distribution web.pdf` and spreadsheet describe a website where
users register, add stock for distribution, provide item details, join item
groups, receive notifications, and contact an owner to request the required
amount.

The original fields include item name, category, quantity, unit price, expiry
date, distribution reason, distribution type, photos, groups, and user location.

## Refined Product Direction

ReDist should not work like a simple contact directory. The production platform
needs a controlled request and reservation workflow so inventory quantities,
approvals, cancellations, handovers, disputes, and audit history remain
accurate.

The recommended MVP is:

- Organizations and members.
- Public discovery for published listings.
- Safe non-regulated categories first.
- Draft, publish, pause, request, accept, reserve, cancel, decline, and complete.
- In-platform messaging and notifications after the core flow is stable.
- English-first UI with Arabic and RTL readiness.

## Deferred Until Compliance Is Ready

- Medicines and prescription products.
- Hazardous materials.
- Online payment and escrow.
- Delivery-provider integrations.
- Native mobile apps.
- Enterprise inventory integrations.

## Brand

- Public name: Redistribut.
- Short name: ReDist.
- Domain: `redistribut.com`.
- Recommended first deployment: one responsive app at `redistribut.com`.
- Optional later workspace split: `app.redistribut.com` and `admin.redistribut.com`.
