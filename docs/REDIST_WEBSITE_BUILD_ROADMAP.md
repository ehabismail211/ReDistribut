# ReDist Website Build Roadmap

Date: 2026-06-22  
Status: Marketing website implementation roadmap  
Scope: Documentation-only build plan

## Roadmap Principle

Build the smallest public website that can generate qualified founder conversations.

Do not build a large brand site before:

- Real pilot workflows exist.
- Legal/privacy copy is reviewed.
- Contact handling process is ready.
- Founder can respond to leads.

## Phase 1: Launch Website

Goal:

- Publish a credible public web presence for controlled pilot conversations.

Pages:

- Home.
- How It Works.
- Suppliers.
- Recipients.
- FAQ.
- Contact.

Required assets:

- [Website Sitemap](REDIST_WEBSITE_SITEMAP.md)
- [Homepage Copy](REDIST_HOMEPAGE_COPY.md)
- [How It Works Page](REDIST_HOW_IT_WORKS_PAGE.md)
- [Supplier Landing Page](REDIST_SUPPLIER_LANDING_PAGE.md)
- [Recipient Landing Page](REDIST_RECIPIENT_LANDING_PAGE.md)
- [Public FAQ](REDIST_FAQ_PUBLIC.md)
- [Brand Messaging Guide](REDIST_BRAND_MESSAGING_GUIDE.md)
- [Lead Capture Strategy](REDIST_LEAD_CAPTURE_STRATEGY.md)

Success criteria:

- Website explains ReDist in under 10 seconds.
- Supplier, recipient, and partner paths are clear.
- Contact form captures qualified leads.
- No real pilot results are claimed.
- No audited ESG or legal certificate claims are made.

## Phase 2: Lead Generation

Goal:

- Use the website to support founder outreach and capture qualified inbound interest.

Add:

- Impact page.
- Partners page.
- Better contact segmentation.
- Basic analytics.
- UTM tracking for outreach.
- Lead source tracking.

Required assets:

- [Conversion Funnel](REDIST_CONVERSION_FUNNEL.md)
- [Analytics Plan](REDIST_ANALYTICS_PLAN.md)
- [Founder Outreach Dashboard](REDIST_FOUNDER_OUTREACH_DASHBOARD.md)
- [Partnership Pipeline](REDIST_PARTNERSHIP_PIPELINE.md)

Success criteria:

- Leads can be segmented by supplier, recipient, partner, ESG, demo, or other.
- Founder can see which sources generate qualified conversations.
- FAQ reduces repetitive explanation.
- Contact form data maps into the outreach pipeline.

## Phase 3: Partner Acquisition

Goal:

- Use the website to support strategic partnerships, ESG sponsorship, NGO introductions, and enterprise discovery.

Add:

- Partners page.
- Impact evidence page after real pilot data exists.
- Case study placeholders replaced only with approved real evidence.
- Partnership inquiry workflow.
- Press release page if launch announcement is approved.

Required assets:

- [Partnership Package](REDIST_PARTNERSHIP_PACKAGE.md)
- [Executive Summary](REDIST_EXECUTIVE_SUMMARY.md)
- [Impact Model](REDIST_IMPACT_MODEL.md)
- [Launch Press Release Draft](REDIST_LAUNCH_PRESS_RELEASE.md)

Success criteria:

- Partner inquiries are qualified and routed.
- ESG claims remain evidence-based.
- Approved pilot results can be added without rewriting the site.
- Website supports founder-led partner conversations.

## Recommended Website Launch Sequence

1. Build Phase 1 pages locally.
2. Add contact form or simple protected inquiry path.
3. Add analytics with privacy-conscious defaults.
4. Review all public claims against brand messaging guide.
5. Review contact/privacy language.
6. Deploy to protected preview.
7. Founder reviews desktop and mobile.
8. Soft-launch URL only in founder outreach.
9. Monitor lead quality.
10. Add Phase 2 pages after first qualified conversations.
