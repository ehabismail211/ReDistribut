## Hero

Purpose:

- Explain ReDist in one screen.
- Route qualified visitors to a founder-led conversation.

Recommended headline:

Redistribute surplus resources with trust, evidence, and impact.

Recommended subheadline:

ReDist helps verified UAE organizations move usable surplus from listing to request, handover, certificate, and impact tracking.

CTA:

- Primary: Request a pilot conversation.
- Secondary: See how it works.

Trust note:

ReDist is currently preparing founder-guided UAE pilot workflows. Public impact results will be shared only after real transfers are completed and approved for publication.

## Features

Purpose:

- Explain the practical platform capabilities without sounding like a generic SaaS feature grid.

Feature blocks:

| Feature | Copy |
| --- | --- |
| Resource listings | Suppliers list available surplus with quantity, condition, location, and handover notes. |
| Resource discovery | Recipients discover available resources by practical sourcing details. |
| Request workflow | Recipients submit structured requests and suppliers review before approving. |
| Handover tracking | Approved requests move into transfer and verification workflows. |
| Certificate evidence | Completed transfers can generate operational evidence. |
| Impact visibility | ReDist helps organizations understand value recovered, items redistributed, and waste diverted. |

## How It Works

Purpose:

- Make the workflow easy to understand.

Steps:

1. List.
2. Discover.
3. Request.
4. Transfer.
5. Verify.
6. Impact.

CTA:

- Learn how ReDist works.

Implementation note:

- Use a horizontal timeline on desktop and stacked cards on mobile.

## Trust

Purpose:

- Reduce concern about organization identity, approvals, certificates, and platform responsibility.

Trust points:

- Verified organization workflows.
- Role-based access.
- Request approval before handover.
- Transfer status tracking.
- Certificate-backed completion evidence.
- Impact metrics tied to completed workflows.

Required disclaimer:

ReDist certificates are operational evidence of a platform workflow. They are not legal guarantees, government approvals, safety certifications, tax receipts, or audited ESG certificates.

## Impact

Purpose:

- Explain the impact model without claiming real results.

Metrics to show as capabilities:

- AED value recovered.
- Items redistributed.
- Waste diverted.
- Certificate evidence.
- Recipient usefulness.
- Environmental estimates where defensible data exists.

Boundary copy:

Public impact claims will be based only on completed, verified workflows and clearly labeled estimates.

- Organization type.
- Short description of surplus, need, or partnership idea.
- Consent to be contacted.
