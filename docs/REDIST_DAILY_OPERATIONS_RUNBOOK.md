# ReDist Daily Operations Runbook

Date: 2026-06-21  
Cadence: Every operating day  
Owner: Founder during pilot; Operations Lead after first hire

## Purpose

Use this runbook to keep daily ReDist operations controlled, visible, and safe.

## Daily Checklist

| Step | Area | Owner | Status |
| --- | --- | --- | --- |
| 1 | Dashboard review | Founder / Ops | Pending |
| 2 | Verification review | Founder / Reviewer | Pending |
| 3 | Transfer monitoring | Founder / Ops | Pending |
| 4 | Certificate monitoring | Founder / Ops | Pending |
| 5 | Issue review | Founder / Support | Pending |
| 6 | Participant tracking update | Founder / Ops | Pending |
| 7 | Escalation check | Founder | Pending |

## 1. Dashboard Review

Review:

- Needs Attention.
- Requests awaiting response.
- Transfers awaiting verification.
- Expiring or urgent listings.
- Admin review items.
- Active workflow status.
- Impact snapshot changes.

Questions:

- What requires action today?
- Which participant is blocked?
- Which workflow could become a trust or timing issue?
- Is anything urgent enough to contact a participant now?

Output:

- Daily priority list.
- Updated tracking center.

## 2. Verification Review

Review:

- New organization verification submissions.
- Expired or near-expiry documents.
- Category eligibility.
- Reviewer notes.
- Documents needing correction.

Actions:

- Approve low-risk complete submissions.
- Request clarification where needed.
- Keep restricted categories disabled until properly reviewed.
- Escalate unclear legal/category issues to founder/legal review.

Output:

- Verification decisions recorded.
- Issue log entry for any blocker.

## 3. Transfer Monitoring

Review:

- Accepted requests ready for handover.
- Transfers awaiting verification.
- Delayed or blocked handovers.
- Quantity/location mismatches.
- Participant messages or notes.

Actions:

- Confirm next step owner.
- Contact supplier/recipient if timing is unclear.
- Pause completion if facts are disputed.
- Preserve evidence if dispute begins.

Output:

- Updated transfer status.
- Escalations assigned.

## 4. Certificate Monitoring

Review:

- Certificates generated after completed transfers.
- Certificate references in tracking center.
- QR verification status.
- Certificate disputes or mismatches.
- Public-safe certificate output.

Actions:

- Confirm certificate exists for each completed transfer.
- Review first certificate for each participant before external sharing.
- Pause sharing if data is disputed.
- Record certificate ID in tracking center and success metrics.

Output:

- Certificate status recorded.
- Issues logged if mismatch exists.

## 5. Issue Review

Review:

- New issues.
- Open Critical or High issues.
- Aging Medium issues.
- Repeated Low issues that indicate training/product friction.
- Participant feedback forms.

Actions:

- Assign owner and severity.
- Pause affected workflow for Critical issues.
- Resolve or document workaround for High issues.
- Update weekly review notes.

Output:

- Issue log current.
- Escalation owner assigned.

## Daily Closeout

Before ending the day:

- Update `docs/UAE_PILOT_WAVE1_TRACKING_CENTER.md`.
- Update `docs/UAE_PILOT_ISSUE_LOG.md`.
- Update `docs/UAE_PILOT_SUCCESS_METRICS.md` if a metric changed.
- Confirm no unresolved Critical issue is unowned.
- Confirm next participant action.

## Daily Red Flags

Escalate immediately if:

- Cross-organization data exposure is suspected.
- Unsafe/restricted goods appear.
- Certificate data is wrong.
- Transfer is marked complete but either party disputes it.
- Private document/evidence may be exposed.
- Founder cannot identify next owner for a blocked workflow.
